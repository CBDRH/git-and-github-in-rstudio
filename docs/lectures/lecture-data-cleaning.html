Keep rows that match a condition with filter()

# Load dplyr library
library(dplyr)

# Load cars dataset
data(cars)

# Filter rows
cars |> 
  filter(speed == 10)
  speed dist
1    10   18
2    10   26
3    10   34


# Filter rows
cars |> 
  filter(speed >= 10 & dist <= 20)
  speed dist

The pipe operator |>

# Sort by speed
cars |> arrange(speed)


  • |> is called the pipe operator
  • It takes the object on the left and passes it to the first argument of the function on the right


# Equivalent to above code snippet
arrange(cars, speed)


  • We can use |> to chain multiple dplyr verbs
  • keyboard shortcut is Ctrl/Cmd + Shift + M

The old pipe operator %>%

If you‚Äôve been using R for a while you may be familiar with the %>% pipe operator shipped with the magrittr package.


# Exactly the same thing!
cars |> 
  mutate(time = dist / speed)  |> 
  arrange(-time)  |>  
  slice(1)
# Exactly the same thing!
cars %>% 
  mutate(time = dist / speed) %>%
  arrange(-time) %>% 
  slice(1)


  • For simple cases, |> and %>% behave the same
  • |> is simpler
  • |> is implemented in base R

A note on style

# Data prep
cars |> 
  mutate(time = dist / speed) |> 
  arrange(-time) |> 
  slice(1)
  • Neat and easy to read!
  • Each verb on a new line
  • |> at end of line
  • Indentation
  • Use of spacing

Avoid this!

# Data prep
cars|>mutate(time=dist-speed)|>arrange(-time)|>slice(1)

Factors in R

We can group_by() this new variable, then use summarise() to calculate the average speed for each group.

# Create a new variable
cars |> 
  mutate(
    class = case_when(
      speed <= 4 ~ "slow", 
      speed <= 10 ~ "average", 
      .default = "fast"
    )) |> 
  group_by(class) |>  
  summarise(
    mean = mean(speed)
  ) 
# A tibble: 3 √ó 2
  class    mean
  <chr>   <dbl>
1 average  8.71
2 fast    17.1 
3 slow     4

Notice the ordering of the output! This is because the new class variable is a character vector.

Factors in R

Factors are useful for handling categorical variables.

  • Category names
  • Ordering categories
  • Mapping colors to categories (later!)

To create a factor you must start by creating a list of the valid levels:

# Vector of valid levels
classLevels <- c('slow', 
                 'average', 
                 'fast')

# Create factor variable
factor(c('fast', 'slow'), 
       levels = classLevels) 
[1] fast slow
Levels: slow average fast

Factors in R

Let‚Äôs put it all together!

# Create a new variable
cars |> 
  mutate(
    class = 
      factor(
        case_when(
          speed <= 4 ~ "slow", 
          speed <= 10 ~ "average", 
          .default = "fast"),
        levels = classLevels)) |> 
  group_by(class) |>  
  summarise(
    mean = mean(speed)
  ) 
# A tibble: 3 √ó 2
  class    mean
  <fct>   <dbl>
1 slow     4   
2 average  8.71
3 fast    17.1


Now the ordering is by speed class, not alphabetically!
